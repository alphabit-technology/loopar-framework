Server-Side Rendering
---------------------

References: `[src-webpack/server](https://github.com/alphabit-technology/loopar-framework/blob/50f82c7762406505292965a27a100bcedf75517c/src-webpack/server)`

The `[â¦/server](https://github.com/alphabit-technology/loopar-framework/blob/50f82c7762406505292965a27a100bcedf75517c/src-webpack/server)` directory implements server-side rendering (SSR) for the Loopar framework. SSR allows rendering the initial React markup on the server, avoiding an initial blank page and improving performance by reducing the need for client-side JavaScript execution.

The main `/server` directory contains functionality to handle rendering. It renders the top-level component to a string, passing props.

If redirects need to be handled after rendering, an object is returned. Otherwise an object containing the rendered HTML string is returned. This allows redirects to be handled on the server.

The `/middleware` subdirectory implements critical server-side middleware functionality. It uses the rendering to pre-render the React app string on each request. When the request comes in, it renders the app and returns the pre-rendered HTML, avoiding an initial blank page.

In summary, the SSR functionality allows Loopar to render the initial React markup on the server for fast page loads. It leverages React and middleware to enable pre-rendering and redirection handling before sending the HTML response.
